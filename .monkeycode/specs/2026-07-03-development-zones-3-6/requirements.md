# 3-6 岁八大发展专区需求

## 1. 目标

小牛育儿 5.0 需要新增 3-6 岁发展专区，把语言发育、感统支持、专注力、大运动练习、情绪管理、社交能力、自信与适应、生活习惯整合成可观察、可练习、可记录、可追问的家庭支持体系。

## 2. 需求

### R1 专区入口

- R1.1 小程序应在首页展示 3-6 岁发展专区入口。
- R1.2 专区入口应聚焦 3-4 岁、4-5 岁、5-6 岁。
- R1.3 专区应包含语言发育、感统支持、专注力、大运动练习、情绪管理、社交能力、自信与适应、生活习惯 8 个方向。

### R2 专区内容

- R2.1 每个专区应按家长能理解的场景组织内容。
- R2.2 每个场景应提供一个今日可做的短练习。
- R2.3 每个专区应支持按 3-4 岁、4-5 岁、5-6 岁展示不同内容。

### R3 首页体验

- R3.1 首页应新增“3-6 岁发展专区”模块。
- R3.2 每个专区卡片应展示标题、场景化副标题和行动提示。
- R3.3 首页应保留成长观察、每日练习、育儿锦囊、小牛问答和成长记录入口。
- R3.4 首页专区卡片应适配小屏，避免文字挤压和布局跳动。
- R3.5 新模块应保持当前小牛育儿视觉体系。

### R4 专区详情

- R4.1 用户点击专区后应进入专区详情页。
- R4.2 专区详情页应支持年龄段切换。
- R4.3 如果当前孩子有生日，详情页应自动选中对应 3-6 岁年龄段。
- R4.4 专区详情页应展示表现选择、今日练习、7 天计划、相关锦囊、问小牛、记录变化入口。
- R4.5 缺少生日时不应默认到错误年龄段，应让用户选择年龄段或补档案。
- R4.6 页面跳转应保持现有小程序路由风格。

### R5 场景与练习

- R5.1 用户应能查看某个场景的详情。
- R5.2 场景详情应说明孩子表现、家长先判断什么、今天做什么。
- R5.3 每个练习应有时长、动作、家长话术和观察指标。
- R5.4 场景页应能带着专区、年龄段和场景上下文跳转小牛问答。
- R5.5 场景页应能把练习记录到成长记录。

### R6 后端数据

- R6.1 后端应提供专区列表和详情数据。
- R6.2 前端和后端的专区 code、年龄段和场景 code 应保持一致。
- R6.3 后端应能按专区、年龄段和场景关联相关文章或练习内容。

### R7 记录闭环

- R7.1 成长记录应支持专区练习来源。
- R7.2 首页今日任务应能承接专区练习。
- R7.3 周总结应能展示专区练习沉淀。

### R8 文案与异常状态

- R8.1 专区文案应使用家庭场景表达专业概念。
- R8.2 缺内容、缺生日、功能未开启等状态应有自然提示。
