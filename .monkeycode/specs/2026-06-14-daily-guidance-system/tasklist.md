# 小牛育儿日常陪伴闭环开发任务清单

**文档日期**: 2026-06-14  
**原则**: 每一步开发都必须可回测、可回滚、可复盘。

## 一、阶段总览

### 阶段1: 今日育儿计划

目标：让首页首屏从静态建议升级为真实可执行的每日计划。

任务：

1. 新增 `daily_plan_records` 与 `daily_plan_completions` 表。
2. 设计首期轻量推荐规则，按年龄、测评短板和最近行为输出 `3` 张计划卡。
3. 开发 `GET /api/v1/daily-plan` 接口。
4. 开发 `POST /api/v1/daily-plan/complete` 接口。
5. 首页接入计划卡展示、完成状态和跳转逻辑。
6. 补齐 `daily_plan_view`、`daily_plan_click`、`daily_plan_complete` 埋点。

回测：

- 验证有孩子且有测评数据用户返回差异化计划。
- 验证有孩子但无测评数据用户返回默认计划。
- 验证无孩子档案用户返回引导卡。
- 验证卡片点击、完成和刷新状态链路。
- 验证接口失败时首页兜底展示。

回滚点：

- 新表独立创建，保留但不影响旧链路。
- 首页支持回退到静态建议文案。

### 阶段2: 成长记录

目标：让家长每日能快速记录孩子状态。

任务：

1. 新增 `growth_daily_records` 表。
2. 开发单日读取、提交、历史读取和趋势摘要接口。
3. 新增成长记录入口页与历史页。
4. 补齐 `growth_record_open`、`growth_record_submit` 埋点。

回测：

- 验证新增、修改、回显、切换孩子、历史查看。
- 验证连续 `7` 天记录后的趋势读取。
- 验证无孩子、无记录、未登录边界场景。

回滚点：

- 新增页面与接口独立开关控制。
- 提交失败不污染旧页面数据。

### 阶段3: 每周成长总结

目标：让家长看到一周变化和下周建议。

任务：

1. 新增 `weekly_growth_summaries` 缓存表。
2. 开发周总结生成服务和读取接口。
3. 新增周总结展示页。
4. 设计普通用户与会员用户差异化内容。
5. 补齐 `weekly_summary_view`、`weekly_summary_action_click` 埋点。

回测：

- 验证完整记录用户、弱记录用户、无记录用户三类输出。
- 验证会员与普通用户展示差异。
- 验证总结结论与原始数据一致。

回滚点：

- 周总结按缓存表独立存储。
- 规则异常时回退到基础趋势摘要。

### 阶段4: 场景化搜索

目标：让搜索首屏优先给方案而不是只给文章。

任务：

1. 新增 `search_scene_tags`、`search_scene_aliases`、`search_scene_recommendations` 表。
2. 建立首批高频场景词库与别名表。
3. 开发 `GET /api/v1/search/scenes` 和 `GET /api/v1/search/solutions` 接口。
4. 升级搜索页结构，加入场景入口和方案卡。
5. 补齐 `scene_search_exposure`、`scene_search_submit`、`scene_solution_click` 埋点。

回测：

- 高频问题词至少回测 `30` 个。
- 验证别名词、模糊词和年龄过滤。
- 验证方案卡、任务卡、文章卡跳转。
- 验证未命中时兜底回退到现有搜索结果。

回滚点：

- 场景结果接口与旧搜索接口并存。
- 新搜索入口可通过开关停用。

### 阶段5: 会员承接优化

目标：让会员权益表达和实际能力开放一致。

任务：

1. 更新会员中心权益文案。
2. 在今日计划、周总结等触点增加会员承接提示。
3. 保持统一兑换码逻辑与新权益口径一致。

回测：

- 验证普通用户和会员用户看到的说明一致。
- 验证会员跳转路径和权益提示。
- 验证不影响现有兑换码发放逻辑。

回滚点：

- 承接文案与功能开关分离。
- 页面触点支持恢复旧文案。

## 二、开发顺序建议

推荐顺序：

1. 先做今日育儿计划。
2. 再做成长记录。
3. 再做每周成长总结。
4. 再做场景化搜索。
5. 最后做会员承接优化。

原因：

- 首页计划最直接影响首屏感知和日常打开率。
- 成长记录和周总结天然构成留存闭环。
- 场景化搜索建立在已有内容和推荐关系之上，适合后置。

## 四、每轮开发的固定要求

每轮开始前必须完成：

- 建立 Git 检查点
- 明确影响范围
- 明确回滚方式

每轮结束后必须完成：

- 本地语法检查
- 接口自测
- 页面联调
- 回测结果留档
- 推送远端仓库

## 五、上线前总回测

上线前必须完成以下总回测：

1. 首页计划全链路回测。
2. 记录与周总结闭环回测。
3. 搜索高频问题命中回测。
4. 会员触点回测。
5. 埋点数据抽样核对。
6. 生产备份与回滚方案确认。
7. 与 `我赢AI` 隔离回归验证。
