# 阶段1隔离部署检查清单

**文档日期**: 2026-06-13  
**用途**: 确保小牛育儿后台后续开发与部署不影响现有我赢AI后台。

## 一、隔离目标

小牛育儿后台必须做到：

- 独立访问入口
- 独立运行进程
- 独立部署目录
- 独立后台表
- 独立回滚路径

## 二、推荐命名

### 2.1 域名

- 后台前端推荐：`admin-niuniu.woyai.cn`
- 后台预发推荐：`admin-niuniu-staging.woyai.cn`

### 2.2 路由前缀

- 后台 API 推荐：`/admin-api/v1`
- 如果需进一步收紧，可使用：`/niuniu-admin-api/v1`

### 2.3 部署目录

- 后台前端：`/home/ubuntu/niuniu-parenting-admin/web`
- 后台 API：`/home/ubuntu/niuniu-parenting-admin/api`

### 2.4 PM2 进程名

- `niuniu-admin-web`
- `niuniu-admin-api`

### 2.5 数据表前缀

- `admin_users`
- `admin_audit_logs`
- `admin_runtime_configs`
- `admin_daily_*`

## 三、Nginx 检查项

上线前必须确认：

1. 小牛育儿后台使用独立 `server_name`。
2. 不修改我赢AI后台现有 `server_name`。
3. 不复用我赢AI后台现有 `location` 规则。
4. 后台静态资源和后台 API 分别指向小牛育儿后台目录或进程。

## 四、PM2 检查项

上线前必须确认：

1. 不重启我赢AI后台 PM2 进程。
2. 小牛育儿后台使用新进程名。
3. 小牛育儿后台日志独立输出。

## 五、数据库检查项

上线前必须确认：

1. 管理后台新增表使用 `admin_` 前缀。
2. 不向我赢AI后台表写入数据。
3. 小牛育儿后台配置和审计数据写入小牛育儿自己的管理表。

## 六、回滚检查项

回滚时必须确认：

1. 只回滚小牛育儿后台站点和进程。
2. 不回滚我赢AI后台站点。
3. 不重启我赢AI后台进程。
4. 数据库回滚只作用于新增 `admin_` 表或新增配置版本。

## 七、阶段1验收

阶段1完成时应确认：

1. 小牛育儿后台命名规范已经固定。
2. 小牛育儿后台域名方案已经确定。
3. 小牛育儿后台部署目录已经确定。
4. 小牛育儿后台 PM2 名称已经确定。
5. 小牛育儿后台与我赢AI后台隔离策略已经通过人工审查。
