# 小牛育儿管理后台开发任务清单

**文档日期**: 2026-06-13  
**原则**: 每一步开发都必须可回测、可回滚、可复盘。

## 一、阶段总览

### 阶段1: 数据与口径底座

目标：让后台数据有统一来源和统一解释。

任务：

1. 盘点现有数据表和现有埋点事件。
2. 设计并创建后台所需新表。
3. 标准化事件字段与事件命名。
4. 补齐关键前端埋点上报。
5. 开发按日统计聚合脚本。
6. 确认后台域名、部署目录、PM2 名称和 Nginx 隔离策略。

回测：

- 抽样核对 `event_tracks` 原始数据与聚合结果。
- 抽样核对注册、活跃、付费三类核心指标。
- 验证新表创建后不影响现有小程序链路。
- 验证隔离方案不影响现有我赢AI后台站点和进程。

回滚点：

- 数据库迁移脚本可单独回滚。
- 新增埋点字段以兼容方式上线。

### 阶段2: 后台认证与权限

目标：让后台具有安全入口。

任务：

1. 新增管理员表。
2. 实现管理员登录接口。
3. 实现管理员权限中间件。
4. 实现后台审计日志。

回测：

- 验证未登录访问被拒绝。
- 验证不同角色访问权限差异。
- 验证高风险接口操作被记录到日志。

回滚点：

- 后台 API 路由独立挂载。
- 管理员表与业务用户表隔离。

### 阶段3: 仪表盘与统计 API

目标：让经营数据先可见。

任务：

1. 开发总览 API。
2. 开发用户趋势 API。
3. 开发收入趋势 API。
4. 开发内容排行 API。
5. 开发板块使用排行 API。

回测：

- 用 SQL 抽样对比接口输出。
- 验证时间筛选条件准确。
- 验证空数据区间返回稳定。

回滚点：

- 统计 API 独立模块化提交。
- 聚合查询支持切回原始查询兜底。

### 阶段4: 后台前端首期页面

目标：交付可登录使用的网址后台。

任务：

1. 初始化后台前端工程。
2. 实现登录页。
3. 实现总览页。
4. 实现用户分析页。
5. 实现会员与收入页。
6. 实现内容与板块页。
7. 使用独立后台域名或独立预发域名完成联调。

回测：

- 验证登录流程。
- 验证图表与卡片数据一致。
- 验证日期筛选、分页、排序行为。
- 验证后台新站点访问不影响现有我赢AI后台入口。

回滚点：

- 后台前端独立部署。
- 可先不上线入口，只通过内网或临时域名验证。

### 阶段5: 配置中心与运营控制

目标：让运营能调配置。

任务：

1. 实现首页板块配置。
2. 实现功能开关配置。
3. 实现推荐位配置。
4. 实现配置版本记录。
5. 实现配置回滚。

回测：

- 验证配置修改后生效。
- 验证回滚后恢复上一个版本。
- 验证审计日志记录完整。
- 验证后台配置变更只作用于小牛育儿，不作用于我赢AI。

回滚点：

- 配置改动写入独立表。
- 配置生效前支持预览和二次确认。

### 阶段6: 经营建议中心

目标：让后台能辅助运营判断。

任务：

1. 定义建议规则。
2. 计算高访问低转化内容。
3. 计算高使用低付费功能。
4. 生成每周建议卡片。

回测：

- 验证建议数据来源。
- 抽样核对建议结论是否与真实报表一致。

回滚点：

- 建议中心只读上线。
- 规则引擎与配置中心分离。

## 二、开发顺序建议

推荐顺序：

1. 先做数据与埋点口径。
2. 再做后台 API。
3. 再做后台前端。
4. 最后做配置中心和建议中心。
5. 每个阶段上线前执行一次与我赢AI后台的隔离回归。

原因：

- 数据口径稳定后，页面和接口反复返工会少很多。
- 先做 API 可以让后续后台前端与自动化测试并行推进。
- 配置中心属于高风险模块，适合在只读后台稳定后进入。

## 四、测试与回测清单

### 4.1 接口测试

- 管理员登录成功与失败
- 权限不足拦截
- 仪表盘返回稳定
- 收入统计口径正确
- 内容排行返回正确

### 4.2 数据测试

- 注册数与 `users` 抽样一致
- 活跃数与 `event_tracks` 抽样一致
- 付费数与 `payment_orders`、`user_memberships` 抽样一致
- 板块访问数与事件数据一致

### 4.3 页面测试

- 登录页
- 总览页
- 图表页
- 筛选页
- 配置中心页

### 4.4 回滚测试

- 配置修改后回滚
- 后台 API 模块关闭后不影响小程序主链路
- 管理后台前端下线后业务服务仍正常

### 4.5 优化回测

每一阶段开发完成后必须追加：

1. 性能检查
2. 指标口径复核
3. 权限边界复核
4. 页面易用性复核
5. 回滚能力复核

## 五、经营建议

### 5.1 你最先该盯的 5 个指标

1. 当前有效会员数
2. 每日活跃用户数
3. 试用转付费转化率
4. AI 问答使用人数与付费转化率
5. 能力成长模块访问量与完成率

### 5.2 你最先该看的 3 条链路

1. 首页 -> 能力成长 -> 会员页 -> 支付成功
2. 首页 -> AI 问答 -> 会员页 -> 支付成功
3. 首页 -> 育儿文章 -> 收藏/再次访问 -> 会员页 -> 支付成功

### 5.3 你最先该做的 3 类优化

1. 优化高流量低转化模块的会员承接。
2. 优化高访问低完成模块的任务长度和交互。
3. 优化高频 AI 提问主题对应的内容供给与付费路径。
