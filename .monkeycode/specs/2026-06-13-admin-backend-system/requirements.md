# 小牛育儿管理后台需求文档

**文档日期**: 2026-06-13  
**目标系统**: 小牛育儿管理后台  
**适用范围**: 运营管理、内容管理、会员与收入分析、行为分析、功能调整

## 一、背景与目标

小牛育儿已经具备小程序前台、会员、支付、能力成长、营养、育儿知识、成长观察和 AI 问答等能力。当前项目缺少面向运营和经营管理的独立后台，导致以下问题：

- 无法通过专属网址集中查看注册、活跃、付费、阅读和转化数据。
- 无法通过统一后台调整首页板块、功能开关、内容发布状态和运营策略。
- 无法快速判断哪些模块带来收入，哪些模块带来活跃，哪些模块需要优化或下线。

本次后台建设的核心目标是：

- 建立一个可登录的专属管理后台，支持日常经营管理。
- 建立可持续的业务指标体系，支撑付费增长和内容运营。
- 建立“数据观察 -> 运营动作 -> 回测验证 -> 优化迭代”的闭环。

## 二、业务目标

后台首期必须支撑以下经营动作：

1. 查看有效会员人数、付费订单走势、收入来源和套餐结构。
2. 查看用户活跃度、留存和功能使用分布，识别高价值用户路径。
3. 查看各板块阅读和使用情况，识别高表现内容和低表现内容。
4. 调整运营配置，包括首页板块排序、功能开关、推荐位和内容发布状态。
5. 为后续优化收入提供数据建议，包括会员转化、内容转化和 AI 使用转化。

## 三、用户角色

### 3.1 超级管理员

- 管理管理员账号与权限。
- 查看全部经营数据。
- 修改全局配置和核心开关。
- 执行高风险操作，如下线功能、调整套餐展示、切换实验开关。

### 3.2 运营管理员

- 查看仪表盘、用户数据、内容数据、会员数据。
- 发布或下线内容。
- 调整首页推荐位、活动位、内容排序。
- 查看运营建议与异常告警。

### 3.3 内容管理员

- 管理育儿知识、能力成长内容、营养推荐内容。
- 维护封面、标题、标签、排序、推荐位。
- 查看单篇内容表现。

### 3.4 数据分析管理员

- 查看指标报表、事件趋势、转化漏斗、留存分析。
- 导出数据。

## 四、后台访问形态

后台应以独立 Web 系统形式交付，具备专属网址，例如：

- `https://admin.woyai.cn`

后台必须满足以下隔离约束：

- 不与现有我赢AI后台复用域名入口。
- 不与现有我赢AI后台复用管理路由。
- 不与现有我赢AI后台复用部署目录和运行进程。
- 不与现有我赢AI后台复用数据库管理表。

后台系统由以下部分组成：

- 后台前端页面
- 后台认证体系
- 后台 API
- 统计聚合层
- 运营配置层

服务器登录、日志查看和数据库查询继续保留为运维手段，日常运营使用后台网址完成。

推荐后台入口策略：

- 首选独立子域名，例如 `https://admin-niuniu.woyai.cn` 或 `https://niuniu-admin.woyai.cn`
- 次选独立主机名，例如 `https://admin-parenting.woyai.cn`

首期不建议复用现有我赢AI的后台网址、后台前端工程或后台 Nginx server block。

## 五、核心需求

### 5.1 管理员认证与权限

系统必须支持：

- 管理员账号登录。
- 管理员会话校验。
- 角色权限控制。
- 操作审计日志。

系统应确保：

- 所有后台接口与小程序用户接口隔离。
- 高风险操作支持二次确认。
- 所有配置变更都可追踪到操作人、时间和变更内容。

### 5.2 总览仪表盘

系统必须展示：

- 今日注册用户数
- 累计注册用户数
- 今日活跃用户数
- 7日活跃用户数
- 30日活跃用户数
- 当前有效会员数
- 今日付费订单数
- 今日收入
- 近7日收入趋势
- AI 问答调用次数
- 各核心板块访问次数

系统应支持：

- 日期范围筛选
- 同比和环比展示
- 关键异常提示

### 5.3 用户分析

系统必须支持查看：

- 注册趋势
- 新用户来源分布
- 活跃用户趋势
- DAU、WAU、MAU
- 次日留存、7日留存、30日留存
- 用户生命周期阶段分布
- 会员用户与非会员用户活跃差异

系统应支持按以下维度筛选：

- 时间范围
- 是否会员
- 用户来源渠道
- 是否有孩子档案
- 是否使用过 AI 问答

### 5.4 会员与收入分析

系统必须支持查看：

- 累计付费用户数
- 当前有效付费会员数
- 月卡、季卡、年卡购买人数与占比
- 订单总数、支付成功数、支付失败数
- 每日收入、每周收入、每月收入
- 用户从注册到付费的转化率
- 试用转正式会员的转化率
- AI 使用用户转会员的转化率
- 各板块带来的会员转化情况

系统应支持输出建议：

- 哪类用户更容易转化
- 哪个内容入口带来更多付费
- 哪个页面存在明显流失

### 5.5 板块与内容分析

系统必须支持按板块查看：

- 首页入口点击量
- 育儿知识阅读量与收藏量
- 能力成长访问量、任务完成量、完成率
- 成长观察发起量、完成量、完成率
- 营养模块访问量、食谱详情访问量、收藏量
- AI 问答次数、问题主题分布、会员使用占比

系统必须支持按内容查看：

- 单篇文章浏览量
- 单篇文章点赞量、收藏量、评论量
- 单个能力点访问量与练习触发量
- 单个食谱访问量与收藏量

### 5.6 运营配置

系统必须支持：

- 首页板块排序
- 首页入口显示开关
- 功能开关控制
- 推荐位配置
- 会员套餐展示文案配置
- 内容发布状态控制

系统应支持：

- 配置版本记录
- 一键回滚到上一个稳定版本
- 灰度开关和实验分组字段预留

### 5.7 内容管理

系统必须支持：

- 育儿文章管理
- 能力成长内容管理
- 营养内容管理
- 标签管理
- 发布时间管理
- 上下线控制

系统应支持：

- 查看内容表现数据
- 根据阅读与转化表现给出优先优化建议

### 5.8 埋点与事件分析

系统必须支持：

- 事件类型趋势统计
- 页面访问排行
- 功能点击排行
- 模块使用路径分析
- 关键漏斗统计

首期关键事件必须覆盖：

- 小程序启动
- 首页入口点击
- 文章列表曝光与点击
- 文章详情访问
- 能力成长章节访问
- 能力点详情访问
- 练习开始与完成
- 成长观察开始与完成
- AI 问答发起
- 会员页访问
- 试用激活
- 支付发起与支付成功

### 5.9 经营建议中心

系统应支持自动给出经营建议，例如：

- 能力成长访问高但会员转化低，建议优化会员页承接。
- 某类文章阅读高但收藏低，建议优化标题与行动建议结构。
- AI 问答高频问题集中在某方向，建议补充对应内容专题或付费服务。
- 某板块访问高但完成率低，建议缩短路径或优化交互。

## 六、关键指标定义

### 6.1 用户指标

- 累计注册用户数: `users` 总数
- 今日注册用户数: 当日新增 `users`
- DAU: 当日有行为事件的去重用户数
- WAU: 近7日有行为事件的去重用户数
- MAU: 近30日有行为事件的去重用户数

### 6.2 会员指标

- 累计付费用户数: 有成功支付且成功激活会员的去重用户数
- 当前有效会员数: 当前会员有效期大于当前时间的用户数
- 试用转化率: 激活试用后最终成为付费会员的用户占比

### 6.3 内容指标

- 阅读量: 内容详情访问事件数
- 收藏率: 收藏用户数 / 浏览用户数
- 完成率: 完成事件数 / 开始事件数

### 6.4 转化指标

- 首页到会员页转化率
- 文章阅读到会员页转化率
- AI 问答到会员页转化率
- 试用到付费转化率

## 七、非功能需求

系统必须满足：

- 后台与小程序业务接口隔离
- 高权限操作可审计
- 所有统计口径可追溯
- 支持分页、筛选、导出
- 支持回滚和稳定版本恢复

系统应满足：

- 首屏可在 3 秒内打开主要仪表盘
- 大部分统计接口在 2 秒内返回
- 聚合统计支持缓存

## 八、分阶段交付目标

### 阶段1: 最小经营后台

范围：

- 管理员登录
- 总览仪表盘
- 付费用户统计
- 活跃用户统计
- 各板块访问统计
- 内容排行

验收标准：

- 可通过后台网址登录
- 能看到核心经营指标
- 能按日期范围筛选

### 阶段2: 运营控制后台

范围：

- 首页配置
- 功能开关管理
- 推荐位管理
- 内容上下线管理
- 配置审计与回滚

验收标准：

- 运营人员可通过后台完成主要配置操作
- 所有配置变更有日志和回滚能力

### 阶段3: 增长与优化后台

范围：

- 留存分析
- 转化漏斗
- 经营建议中心
- A/B 实验预留

验收标准：

- 能识别高价值板块、低转化链路和重点优化方向

## 九、回测要求

每一步开发都必须具备回测能力。

每个阶段交付后必须至少完成：

- 接口回归测试
- 权限回归测试
- 统计口径核对测试
- 配置回滚测试
- 关键页面操作回归测试
- 经营数据抽样核对

## 十、成功标准

后台首期上线后，项目应具备以下管理能力：

- 每天查看收入、付费、活跃和内容表现
- 每周识别高表现板块和低表现板块
- 每周根据数据调整首页展示与运营重点
- 每月复盘会员增长、活跃留存和内容转化
